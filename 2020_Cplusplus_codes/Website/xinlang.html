  3. 删除线：除了Bold、underline、italic还有textline：Ctrl+T

  7. 插入代码块：重要的放在最后面，Ctrl+Shift+L

这里我们使用windows命令行工具，Cygwin和MinGW都包含编译器GUN C++，他们要使用的编译器名为g++

准备工作

  1. 安装好vscode
  2. 安装好C/C++相关拓展程序（如code runer一些配置）
  3. 安装好GCC/Clang并配置好环境（教程） <u>关键步骤：复制MINGW的目录，此电脑>属性>高级系统设置>环境变量>path>添加刚刚复制的目录</u> 验证方式：cmd>g++ -v
  4. 配置好默认生成任务：Tasks:Configure Default Build Task，注意环境变量中要设置每种语言的执行路径

补充知识，充个电

  1. 编译器是把源代码变成可执行文件的，编辑器是你打字的软件。记事本就是一个编辑器，VSC也是编辑器。编辑器是无法编译运行程序的，因为那是编译器的工作
  2. MinGW是gcc在Windows下的移植，gcc是世界上最流行的C/C++编译器组合。但gcc这个名字也指编译C语言的那个程序，g++才是C++编译器。即gcc程序和g++程序包含在gcc套件以及MinGW里，当只说gcc时要根据语境自己区分
  3. 其实MinGW和MinGW-w64只是名字像，它们是两个不同的项目。为了方便，本文中的MinGW指的其实都是MinGW-w64。<del>MinGW本身已经很久没有更新了，不使用它</del>哎呀，原来MinGW是活着的，但它只能产生32位程序
  4. 扩展是extension，插件是plugin，VSC用的是前者这种称呼。大部分文章都是混用两者的，不严谨但是能理解就行，要学会抓主要矛盾。当然本文用的都是正确的

菜单项

我们创建好一个项目，需要对程序进行编译和链接，IDE（integerated development environment）通常提供了多个菜单项，比如Compile（编译），Build（建立），Make（生成），Link（链接），Execute（执行），Run（运行）和Debug（调试）

  1. compile：当前在修改哪一个文件，就会编译哪一个文件，生产.obj文件
  2. Build通常意味着编译项目中所有源代码文件的代码。Build=compile+link，生成可执行.exe 文件
  3. Make和build类似，将修改过的文件编译后和所有文件链接
  4. Link：将编译后所有的源代码与所需的库代码（比如一些头文件？）组合起来
  5. Debug：以步进方式执行程序

  8. 链接：将目标代码和C++函数库相连接，并将源程序所用的库代码与
  9. 执行：在特定的机器环境下运行C++程序
